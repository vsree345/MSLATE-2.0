## 1. Project Overview

`TrapX_agent.py` is a **historical replay engine** that reads 1-minute OHLC candle data and options OI data, then applies four institutional-grade intraday trade strategies minute-by-minute, exactly as a human analyst would follow the rules from the `WorkFlow Enhancements.xlsx` plan.

**What it does:**
- Reads historical CSV data (spot OHLC + options OI + VIX)
- Processes every 1-minute candle in sequence (09:15 → 15:29)
- Detects LIS candles, PDL breaks, Fibonacci levels, OI squeezes
- Generates trade signals (CALL / PUT entries, Pyramid top-ups)
- Applies signal-ignore rules (Divergence+1, Squeeze, Green LIS)
- Saves all signals to a timestamped CSV

**What it does NOT do:**
- Live trading or broker API calls
- Real-time streaming
- Stop-loss / target management (just entry signals)

## 4. Core Concepts & Abbreviations

These match the definitions in the `WorkFlow Enhancements.xlsx → "Trades - LifeCycle"` sheet.

| Abbreviation | Full Name             | Definition used in code |
|--------------|-----------------------|-------------------------|
| **Exp Move** | Expected Move         | `spot × (VIX/100) × √(1 / (252×375))` — minimum meaningful price change per 1-min candle |
| **LIS**      | Line in the Sand      | Candle with body > 85% of its range AND body > Expected Move |
| **PDL**      | Previous Day Low      | Prior session's lowest traded price |
| **PDH**      | Previous Day High     | Prior session's highest traded price |
| **PDC**      | Previous Day Close    | Prior session's closing price |
| **Pyramid**  | Top-Up                | Adding to an existing winning position |
| **OI**       | Open Interest         | Number of outstanding option contracts |
| **Jerk**     | % OI Change           | `(current_OI − prev_OI) / prev_OI × 100` |
| **Squeeze**  | OI Divergence         | ITM OI making negative % change AND same-side OTM OI making positive % change |
| **ATM**      | At The Money          | Strike closest to current spot price |
| **ITM**      | In The Money          | CE: strike < spot; PE: strike > spot |
| **OTM**      | Out Of The Money      | CE: strike > spot; PE: strike < spot |
| **CE**       | Call instrument       | Bullish options contract |
| **PE**       | Put instrument        | Bearish options contract |
| **Fibo**     | Fibonacci Retracement | Standard 23.6%, 38.2%, 50%, 61.8%, 78.6% levels |
| **Divergence+1** | OI Divergence Positive | OTM opposite-side OI increasing while price makes negative signal — bullish / bearish confirmation |

### How Expected Move is Calculated

```
Exp Move = Spot × (VIX / 100) × √( 1 / (252 × 375) )
```

- 252 = trading days per year
- 375 = trading minutes per session (09:15 to 15:29)

Example: Spot = 22,500, VIX = 14.5 → **Exp Move ≈ 10.6 pts/min**

| Column      | Type      | Description                      | Example                  |
|-------------|-----------|----------------------------------|--------------------------|
| `timestamp` | datetime  | Candle open time (1-min bar)     | `2026-02-20 09:15:00`    |
| `open`      | float     | Opening price                    | `22501.35`               |
| `high`      | float     | Candle high                      | `22515.72`               |
| `low`       | float     | Candle low                       | `22497.10`               |
| `close`     | float     | Closing price                    | `22508.60`               |
| `volume`    | int       | Traded volume (optional)         | `48000`                  |

**Requirements:**
- Timestamps must be `YYYY-MM-DD HH:MM:SS` format
- One row per minute, session 09:15 to 15:29 (375 rows per day)
- Must be sorted ascending by timestamp

### 5.2 `options_oi.csv` — Options OI Per Strike

| Column            | Type      | Description                                          | Example          |
|-------------------|-----------|------------------------------------------------------|------------------|
| `timestamp`       | datetime  | Same 1-min timestamp as spot candle                  | `2026-02-20 09:15:00` |
| `strike`          | float     | Option strike price (multiples of strike_interval)   | `22500`          |
| `instrument_type` | str       | `CE` or `PE`                                         | `CE`             |
| `oi`              | int       | Raw open interest (number of contracts)              | `485000`         |
| `oi_change_pct`   | float     | % change in OI from previous timestamp               | `-2.4500`        |
| `ltp`             | float     | Last traded price of this option (optional)          | `142.50`         |

**Requirements:**
- One row per (timestamp, strike, instrument_type) combination
- Cover strikes from roughly `spot − 400` to `spot + 400`
- `oi_change_pct` is percentage (e.g., `-2.45` means OI decreased by 2.45%)

### 5.3 `vix_data.csv` — Daily VIX

| Column | Type     | Description                | Example      |
|--------|----------|----------------------------|--------------|
| `date` | date     | Trading date               | `2026-02-20` |
| `vix`  | float    | India VIX closing value    | `14.50`      |

## 7. The Four Strategies

### Strategy 1 — Bounce with PDL Break

**When:** Pre-market / opening session (09:15–10:00 area)
**Direction:** UP (CALL)
**Trade Type:** Initial

**Logic (3 phases):**

```
Phase A  [09:29]  PDL is broken — previous candle's close < PDL
                  → This is the "trigger" candle

Phase B  [09:30]  Next candle must be a LIS (green, body > 85% of range,
                  body > 2× expected_move) — confirms bottom is forming
                  → LIS candle is registered as LIS_BOTTOM

Phase C  [09:31+] Wait for price to retrace back and test the LIS high
                  as support (candle.low ≤ LIS_high + tolerance)
                  → ENTRY: Initial CALL
```

**Ignore rule:** If -ve jerk appears at 09:31–09:36, ignore it if OTM PE OI is increasing (Divergence+1).

---

### Strategy 2 — LIS High Tested in Retracement (Pyramid)

**When:** After an existing CALL position is open
**Direction:** UP (adds to CALL)
**Trade Type:** Pyramid (Top-Up)

**Logic:**

```
Prerequisite: position = CALL (Strategy 1 must have already entered)

Conditions:
  1. A prior LIS candle's high exists in state
  2. Price RALLIED above that LIS high after it formed
  3. Price RETRACED back down to test the LIS high as support
  4. Either:
     a. ATM CE OI increasing OR OTM PE OI increasing (institutional support)
     b. -ve signal can be ignored (CE squeeze, OTM PE divergence, green LIS)
  5. Cooldown: 15 minutes since last pyramid
  6. Each LIS level can only be pyramided once
```

**Ignore rules at 10:15–10:20:**
- 10:13 -ve jerk: ignored because CE squeeze in trade direction
- 10:16–10:18 -ve LIS signals: ignored because green LIS candle present AND squeeze

---

### Strategy 3 — Finding Trade at Top with Less SL

**When:** After 12:30, when price is near session high
**Direction:** DOWN (PUT)
**Trade Type:** Initial, with tight stop-loss

**Logic:**

```
Guards:
  - Time must be >= 12:30
  - Price within 5× expected_move of session high
  - At least 60 candles of history

Conditions:
  1. 2+ upper wick rejection candles in last 10 bars
     (upper_wick > body × 1.5 AND upper_wick > 0.5× expected_move)
  2. PE Squeeze: ITM PE OI change < 0  AND  OTM PE OI change > 0
  3. Negative signal: ATM CE OI change < 0

Entry: Initial PUT
Stop Loss: Just above the highest rejection wick high
```

**Reference candles (20-Feb-2026):**
- 12:25, 12:30 → upper wick rejections (big spike up, close lower)
- 12:44 → PE squeeze detected
- 12:46–12:48 → PUT entry with SL above rejection wick

---

### Strategy 4 — Fibonacci Retracement from Bottom

**When:** Afternoon session (post 13:00), after a midday retracement
**Direction:** UP (CALL)
**Trade Type:** Initial

**Logic:**

```
Setup:
  - Swing low  = Most recent LIS_BOTTOM high (confirmed support level)
  - Swing high = Morning session peak (highest candle high before 12:30)
  - Calculate Fibonacci levels from swing_low to swing_high

Conditions:
  1. len(history) >= 30 candles
  2. Recent green LIS candle in last 4 candles
  3. Price near 50% Fibonacci level (current candle OR the LIS candle's low)
  4. Either:
     a. CE Squeeze (ITM CE OI < 0 AND OTM CE OI > 0)
     b. OTM PE OI increasing (supports upside)

Entry: Initial CALL
Ignore: -ve signals when OTM PE OI is supporting
```

**Reference (20-Feb-2026):**
- 13:10 → Near 50% Fibo
- 13:13 → LIS upside candle (+85 pts green)
- 13:14 → OTM PE OI increasing → enter CALL, ignore -ve signals
- 13:25 → 13:23 -ve signal ignored (OTM PE supporting)

## 8. Signal Filtering Logic

The `should_ignore_negative_signal()` function is the core filter. It is called whenever a -ve jerk or signal appears after entry.

| Rule | Condition | Direction | Reason to Ignore |
|------|-----------|-----------|-----------------|
| **Divergence+1** | OTM PE OI increasing | UP trade | Market participants building puts further OTM = expects upside |
| **Divergence+1** | OTM CE OI increasing | DOWN trade | Building calls further OTM = expects downside |
| **Squeeze (UP)** | ITM CE OI < 0 AND OTM CE OI > 0 | UP trade | CE squeeze — shorts covering ITM, building OTM = bullish |
| **Squeeze (DOWN)** | ITM PE OI < 0 AND OTM PE OI > 0 | DOWN trade | PE squeeze — shorts covering ITM PE, building OTM PE = bearish |
| **Green LIS** | Green LIS candle in last 5 bars | UP trade | Strong directional momentum candle overrides -ve jerk |

When a signal is ignored, it is recorded in `state.ignored_signals` and printed in the run summary.

## 9. Output Format

### Console Log
Every signal prints a line like:
```
[HH:MM:SS] STRATEGY_NAME              | trade_type               | message
```

### `data/trade_signals_YYYY-MM-DD.csv`

| Column           | Description                                        |
|------------------|----------------------------------------------------|
| `strategy`       | Strategy name (`Bounce_PDL_Break`, etc.)           |
| `time`           | Time of signal (HH:MM:SS)                          |
| `direction`      | `UP` or `DOWN`                                     |
| `trade_type`     | `Initial - CALL`, `Pyramid - CALL`, `Initial - PUT` |
| `entry_price`    | Candle close at entry (blank for confirmation rows) |
| `phase`          | Internal phase (`LIS_BOTTOM_CONFIRMED`, `ENTRY`, `PYRAMID_ENTRY`) |
| `retracement`    | `True` if this is a retracement entry              |
| `lis_ref_time`   | Time of the LIS candle this entry references       |
| `ignore_neg_signal` | `True` if a -ve signal was overridden           |
| `ignore_reason`  | Text reason for ignoring -ve signal                |
| `sl`             | Stop loss price level (Strategy 3 only)            |
| `sl_distance`    | Points from entry to SL (Strategy 3 only)          |
| `fibo_level`     | Fibo level used (Strategy 4 only: `50%`)           |
| `fibo_price`     | Calculated Fibo 50% price level                    |
| `swing_low`      | Swing low used for Fibo calculation                |
| `swing_high`     | Swing high used for Fibo calculation               |
| `ce_squeeze`     | `True` if CE squeeze detected (Strategy 4)         |
| `otm_pe_support` | `True` if OTM PE OI was supporting                |
| `message`        | Human-readable signal description                  |

## 12. Tuning Parameters

All tuning constants are inline in the code. Here is a reference for what to adjust:

### In `TrapX_agent.py`

| Location | Parameter | Default | What it controls |
|---|---|---|---|
| `is_lis_candle()` | `body_pct > 0.85` | 85% | Minimum body-to-range ratio for LIS detection |
| `is_lis_candle()` | `body > expected_move` | 1× EM | Minimum body size relative to VIX move |
| `strategy_bounce_pdl_break()` Phase B | `body > 2 * expected_move_1min` | 2× EM | LIS confirmation size threshold |
| `strategy_bounce_pdl_break()` Phase C | `tol = expected_move * 0.8` | 0.8× EM | Tolerance for "support tested" check |
| `strategy_lis_high_retracement()` | `tol = expected_move * 1.2` | 1.2× EM | Tolerance for pyramid retracement test |
| `strategy_lis_high_retracement()` | cooldown `< 15 * 60` | 15 min | Min gap between pyramid top-ups |
| `strategy_lis_high_retracement()` | `post_lis[-15:]` | 15 bars | Lookback to confirm "was above LIS" |
| `strategy_top_trade_less_sl()` | `day_high - close > EM * 5` | 5× EM | Max distance from day high to consider a "top trade" |
| `strategy_top_trade_less_sl()` | `t < time(12, 30)` | 12:30 | Earliest time Strategy 3 can fire |
| `strategy_top_trade_less_sl()` | `len(rejections) >= 2` | 2 | Minimum wick rejection candles needed |
| `strategy_fibo_retracement()` | `abs(lis.low - fib_50) <= tol * 1.5` | 1.5× tol | LIS proximity to Fibo 50% |
| `strategy_fibo_retracement()` | `history[-4:]` | 4 bars | Lookback for recent LIS upside candle |

## 13. Troubleshooting

### "No trade signals generated"

1. Check that `data/spot_ohlc.csv` has data for the correct date.
2. Verify PDL is set correctly — if PDL is wrong, Strategy 1 never triggers.
3. Check that the OI data has the correct time window overlapping with spot data.
4. Add a debug print inside `strategy_bounce_pdl_break()` to trace execution.

### "UnicodeEncodeError" on Windows

This happens if the console encoding doesn't support special characters.
Fix: Run the script with UTF-8 output:

```bash
set PYTHONIOENCODING=utf-8
python TrapX_agent.py
```

### "ModuleNotFoundError: No module named 'openpyxl'"

```bash
pip install openpyxl
```

### Strategy 2 firing too many times

Increase the pyramid cooldown (in `strategy_lis_high_retracement`):

```python
if (curr_dt - last_dt).seconds < 30 * 60:   # 30 min instead of 15
```

Or tighten the tolerance:

```python
tol = state.expected_move_1min * 0.8   # tighter than 1.2
```

### Strategy 3 firing too early

Increase the time gate:

```python
if t < time(13, 0):   # Push to 13:00 instead of 12:30
```

Or require more wick rejections:

```python
if pe_squeeze and negative_signal and len(rejections) >= 3:
```

### Strategy 4 never fires

Check the following:
1. `state.lis_candles` must have a `LIS_BOTTOM` entry (Strategy 1 must run first)
2. Morning candles before 12:30 must exist in history
3. A green LIS candle must appear in the 4 candles before the entry candle
4. Either CE squeeze OR OTM PE OI > 0 at that time

Debug: add a print statement before the `return None` exits to see which condition fails.

### "OI data is empty or stale"

The `build_oi_snapshot()` function takes the latest OI row `<= current timestamp`.
Ensure your `options_oi.csv` timestamps exactly match the spot timestamps.

### Running on BankNifty

Change `strike_interval=100.0` and adjust PDL/PDH accordingly. The LIS and Expected Move thresholds scale automatically based on spot price and VIX.
